# DrawFlow V4 - 开发计划

## 项目概述

**项目名称**：DrawFlow V4
**项目目标**：跨平台 AI 图片生成桌面应用（macOS + Windows）
**技术栈**：Electron + Node.js + Express + Vue 3
**基础版本**：基于 jimeng-web-v2 的 UI 设计

---

## 需求分析

### 核心功能
1. **自定义模板生图** - 上传参考图，输入提示词，生成 AI 图片
2. **多图生成** - 支持一次生成 1-4 张图片
3. **历史记录** - 自动保存生成记录，支持查看和重新生成
4. **OSS 集成** - 参考图和生成图自动上传到阿里云 OSS
5. **@参考图** - 在提示词中快速选择参考图

### 技术要求
1. ✅ 符合 Apple 和 Microsoft 官方规范
2. ✅ 跨平台（macOS + Windows）
3. ✅ 提供两个版本的安装包（.dmg/.zip for macOS, .exe/.zip for Windows）
4. ✅ 极简 UI 设计（复用 jimeng-web-v2 的设计）
5. ✅ 无外部依赖（Python 等）

---

## 技术选型

### 架构方案：Electron + Node.js 后端

| 组件 | 技术选型 | 理由 |
|------|----------|------|
| 桌面框架 | Electron 28 | 跨平台、成熟稳定、官方支持 |
| 后端框架 | Express | Node.js 生态标准、轻量高效 |
| 前端框架 | Vue 3 | 与 jimeng-web-v2 保持一致 |
| 数据库 | better-sqlite3 | 同步 API、性能优秀 |
| 图片处理 | Sharp | Node.js 生态最佳图片处理库 |
| OSS SDK | ali-oss | 阿里云官方 Node.js SDK |
| HTTP 客户端 | axios | 成熟稳定、支持拦截器 |
| 文件上传 | multer | Express 生态标准 |
| 打包工具 | electron-builder | 官方推荐、支持双平台 |

### 为什么选择这个方案？

1. **符合官方规范** ✅
   - Electron 官方推荐使用 Node.js 作为后端
   - 无需 spawn 子进程
   - 符合应用沙箱原则

2. **跨平台支持** ✅
   - electron-builder 原生支持 macOS 和 Windows
   - 一次开发，双平台部署

3. **打包简单** ✅
   - 所有代码打包进 app.asar
   - 无需处理 Python 依赖
   - 支持代码签名和 notarization

4. **维护成本低** ✅
   - 全栈 JavaScript/TypeScript
   - 团队技能统一
   - 生态成熟、文档完善
